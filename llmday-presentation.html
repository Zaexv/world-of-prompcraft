Slide 03 / Why this world makes sense
Design logic

Why a fantasy world is a strong fit for agentic NPCs

Why the setting works
  • Fantasy worlds naturally contain merchants, guards, quest givers, druids, and rivals with clear roles
  • Persistent towns and factions make relationships and memory feel meaningful
  • Magic, weather, rituals, and world events make tool-driven behavior legible to players
Why the simulation works
  • Manifest-driven zones give the world structure instead of one flat map
  • Server authority gives every action a single source of truth
  • World builder systems let both players and agents change the environment
Speaker line

“This world makes sense because the fiction, the mechanics, and the agent model all ask for the same thing: persistent characters inside a shared realm.”

Slide 05 / Architecture
How it is built

The architecture behind the world

Client layer

Three.js game runtime

Terrain, atmosphere, movement, collision, dungeons, combat UI, quests, and world builder tools all live in the client.

Server layer

FastAPI + WebSockets + WorldState

The server handles connections, nearby broadcasts, cleanup, and authoritative state mutation.

Agent layer

LangGraph per NPC

The agent system injects personality, context, memory, and tools into each non-player character.

One-line architecture summary

The client makes the world feel alive, the server makes it consistent, and the agent layer makes the NPCs feel intentional.

Three layers, one coherent runtime

Slide 06 / NPC architecture
LangGraph loop

The NPC loop is deeper than prompt and reply

01

Pre-check

Fast routing handles simple or out-of-band turns before expensive reasoning.

02

Reason

The model sees persona, goal, player state, world context, recent chat, and lore.

03

Act

Tool calls emit pending actions instead of mutating the world directly.

04

Reflect

The NPC updates internal state so the next encounter feels continuous.

Why this matters

This is what turns a model call into a believable world character rather than a stateless answer box.

Slide 08 / State of the art
Why this is interesting now

Where this sits relative to the state of the art

A common AI NPC demo
  • Interesting dialogue but shallow simulation grounding
  • Little or no memory beyond recent chat
  • No authoritative world model behind the conversation
What the frontier is moving toward
  • Persistent memory and identity over many interactions
  • Tool use, planning, and bounded world actions
  • Embodied agents inside shared simulated environments
Why World of Promptcraft matters

It is a strong state-of-the-art direction because it combines memory, tools, server authority, and a coherent 3D world instead of treating AI as a detached conversation surface.

Position the project against the current frontier
