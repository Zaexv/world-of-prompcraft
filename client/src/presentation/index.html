What is World of Promptcraft?

A procedural world, driven by agents

  • Terrain — generated by procedural algorithms from a biome. Endless, never hand-authored.
  • Meshes & buildings — buildings and props are built by LLMs from existing meshes, then placed procedurally.
  • NPCs — body — generated by an LLM from meshes.
  • NPCs — mind — agentic & LLM-driven: they reason, form their own thoughts, decisions and moods, with personality and memory.
  • Interface — the prompt is the only way to play. No menus, no dialogue trees — you simply write.
How did the idea start?

From the state of the art to a living 3D world

  • Skyrim and Minecraft have AI mods — but no fully agentic game had shipped, built end-to-end with agentic coding.
  • Inspired by Stanford's Generative Agents — arxiv.org/abs/2304.03442.
  • It began as a proof of concept and grew into a complete, explorable 3D world — running entirely in the browser.

Pillar 1 — Generative 3D

A generative 3D CLI with Three.js

  • A modular front-end "CLI" — clean modules (UI · World · Scene · Meshes · Network) let each be built to high quality on its own.
  • Biome-driven world — a manifest + biome builder shape infinite 64×64 terrain chunks.
  • Procedural population — as each chunk loads it spawns buildings, props, vegetation and NPCs from a catalog of 200+ meshes.
  • One recipe per item — a single buildMesh(type) builds any of them.
  • Cinematic render — TAA, bloom, IBL, water & sky, LOD — drawn live in the browser.
Pillar 1 — Why Three.js

Three.js, because the world is just code

  • Code-defined geometry — every mesh is plain TypeScript, so Coding Agents read and write it natively. No binary assets, no art pipeline.
  • New meshes & biomes by prompting — an agent authors a building, creature or biome as a class, registers it, and it's in the catalog.
  • The World Manifest is the source of truth — one document holds the zones, biomes, landmarks and every placed item.
  • I place items by prompting — the World Builder turns a prompt into objects dropped into the manifest, with collision and undo.
  • I manage the whole world by prompting — terrain, biomes, meshes and placement, all from natural language.
A Closer Look — The Picture

How each frame is drawn

  • The image is built in layers — first the scene, then a pass that smooths jagged edges, then a pass that makes bright things glow. If a layer can't run, it quietly skips it.
  • Soft shadows and film-like color make it look less "computer-generated."
  • The sky lights the world — a snapshot of the sky is used to light everything realistically, added a moment after the game starts.
  • It watches its own speed — if the frame rate dips, it lowers detail a touch to stay smooth.
  • Less effort on far-away things — distant objects switch to simpler versions and stop casting shadows.
A Closer Look — The World

An endless land, built as you walk

  • The land is made of square tiles (about 64 metres each) that appear around you as you move — so the world is effectively endless.
  • Each tile's hills and valleys come from a formula, not a saved map. Mixing "recipes" gives forests, snow, lava and more.
  • Ground flattens under buildings automatically, so nothing floats or sinks.
  • As a tile appears, the game fills it with trees, rocks and buildings.
  • Every object is a small reusable recipe — defined once, dropped in anywhere with a single line.
  • You can't walk through walls — a fast "bumping" check keeps you on the ground and out of solid things.
Pillar 2 — Agentic Backend

An agentic backend on LangGraph

  • Every character thinks on its own — each has a private decision loop and its own memory; new characters can be added while the game runs.
  • The thinking happens in small steps on the server and is sent back over the live connection.
  • "Guardrails" keep it on track so the AI stays in character and the story stays consistent.
  • A quick "lore lookup" feeds each character the facts it needs, so it can reason in the moment, change mood, and remember.
A Closer Look — Thinking

Six steps, but only one or two AI calls

  • Think — the character gathers who it is, what's nearby, your shared history, its mood, and any relevant lore, then decides what to say and do. Plain "hellos" take a quick shortcut to save effort.
  • Do it — it carries out the chosen actions (attack, give an item…), sees the result, and loops back to think if there's more to do.
  • Reply — it writes the actual in-character line, reusing what it already worked out when that text is clean.
  • Fill gaps — if it somehow says nothing, a sensible line is built from what it just did.
  • Update feelings — no AI here: simple rules adjust its mood and how much it likes you (a score from −100 to +100).
  • Trim memory — once in a while it compresses old chat into a short note, so a character's memory never grows out of control.

Only the first two steps may call the AI — the rest is cheap, ordinary code. That keeps each reply fast and the cost predictable.

A Closer Look — Memory

Characters remember you — no database needed

  • A private notebook per pair — each character keeps a separate memory for each player it has met.
  • What it keeps — a short summary of your chats, its current mood, how much it likes you, and notes it has formed about you.
  • No wasted thinking — if you ask the exact same thing again, it reuses the previous answer instead of asking the AI twice.

A Closer Look — Safety

Characters act through a fixed set of verbs

  • A limited menu of actions — attack, heal, trade, give a quest, change the weather, and a few more. A character can't do anything outside that list. That's the safety net.
  • Nothing happens immediately — each verb just writes down "what should change." The server then applies the whole list at once, in one clean step.
  • The money is real — offering an item only proposes a price; a sale only goes through if you can actually pay; quest rewards are capped so nothing breaks the economy.
  • One place changes the world — keeping every update in a single, guarded step avoids glitches when many things happen at once.
Keeping it fast & fair

One source of truth, never overloaded

fairness

The server owns the world

Every hit point, coin and weather change is decided in one place. Your browser only shows the result — so two players always see the same thing.

responsiveness

Slow thinking never freezes you

While a character is thinking, you can still walk around freely. Rapid clicks are queued, only a handful of AI requests run at once, and each has a time limit — so it never gets swamped.

speed

Instant for the simple stuff

Ordinary hits are settled instantly by simple rules. The AI is saved for the moments that actually need thought.

cost

Cheap way to recall lore

Background facts are found by matching keywords, not by heavy AI search — so each reply stays low-cost and predictable.

Pillar 3 — Agentic Coding

Agentic coding as the way of building

  • One shared rulebook — a couple of guide documents tell every AI coding assistant how things are built here, so the style stays consistent.
  • Nothing ships untested — automated checks and tests must pass before any change is accepted; problems are caught before they reach the project.
  • Two working habits — research first then build, and repeat-until-right loops.
  • Draw it first — sketch the design as simple diagrams before writing any code.
Takeaways & features

Lessons learnt & cool features

lessons learnt

Lessons

Guardrails beat raw randomness. Giving each character its own brain adds personality without adding complexity. And by using the AI sparingly — only where real thinking is needed — the cost stays low and predictable.

Tooling
